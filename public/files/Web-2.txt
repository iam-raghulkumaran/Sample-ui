Stored Procedures (MANDATORY)
CREATE PROCEDURE usp_GetVendors
AS
BEGIN
    SELECT VendorID, VendorName
    FROM VendorInfo
    ORDER BY VendorName;
END;
GO

CREATE PROCEDURE usp_GetJobTitlesByVendor
    @VendorID INT
AS
BEGIN
    SELECT DISTINCT Job_Title
    FROM ShipmentInfo
    WHERE VendorID = @VendorID
    ORDER BY Job_Title;
END;
GO

CREATE PROCEDURE usp_GetShipmentDetails
    @VendorID INT,
    @JobTitle VARCHAR(10)
AS
BEGIN
    SELECT
        ShipmentID,
        Job_Title,
        ChapterID,
        ShipmentDate
    FROM ShipmentInfo
    WHERE VendorID = @VendorID
      AND Job_Title = @JobTitle;
END;
GO

CREATE PROCEDURE usp_UpdateShipmentDate
    @ShipmentID INT,
    @ShipmentDate DATE
AS
BEGIN
    UPDATE ShipmentInfo
    SET ShipmentDate = @ShipmentDate
    WHERE ShipmentID = @ShipmentID;
END;
GO

DBHelper.cs

using Microsoft.Data.SqlClient;
using System.Data;

namespace Web2.DAL
{
    public class DBHelper
    {
        private readonly string _connectionString;

        public DBHelper(IConfiguration configuration)
        {
            _connectionString = configuration.GetConnectionString("DefaultConnection");
        }

        public DataTable ExecuteProcedure(string procName, SqlParameter[] param = null)
        {
            DataTable dt = new DataTable();

            using SqlConnection con = new SqlConnection(_connectionString);
            using SqlCommand cmd = new SqlCommand(procName, con);
            cmd.CommandType = CommandType.StoredProcedure;

            if (param != null)
                cmd.Parameters.AddRange(param);

            SqlDataAdapter da = new SqlDataAdapter(cmd);
            da.Fill(dt);

            return dt;
        }
    }
}


Controllers/HomeController.cs

using Microsoft.AspNetCore.Mvc;
using Microsoft.Data.SqlClient;
using System.Data;
using Web2.DAL;
using Web2.Models;

public class HomeController : Controller
{
    private readonly DBHelper _db;

    public HomeController(IConfiguration configuration)
    {
        _db = new DBHelper(configuration);
    }

    public IActionResult Index()
    {
        var vendors = new List<VendorModel>();
        DataTable dt = _db.ExecuteProcedure("usp_GetVendors");

        foreach (DataRow r in dt.Rows)
        {
            vendors.Add(new VendorModel
            {
                VendorID = Convert.ToInt32(r["VendorID"]),
                VendorName = r["VendorName"].ToString()
            });
        }
        return View(vendors);
    }

    [HttpGet]
    public JsonResult GetJobTitlesByVendor(int vendorId)
    {
        SqlParameter[] p = { new("@VendorID", vendorId) };
        var dt = _db.ExecuteProcedure("usp_GetJobTitlesByVendor", p);

        return Json(dt.AsEnumerable().Select(r => r["Job_Title"].ToString()).ToList());
    }

    [HttpGet]
    public JsonResult GetShipmentDetails(int vendorId, string jobTitle)
    {
        SqlParameter[] p =
        {
            new("@VendorID", vendorId),
            new("@JobTitle", jobTitle)
        };

        var dt = _db.ExecuteProcedure("usp_GetShipmentDetails", p);
        var list = new List<ShipmentModel>();

        foreach (DataRow r in dt.Rows)
        {
            list.Add(new ShipmentModel
            {
                ShipmentID = (int)r["ShipmentID"],
                Job_Title = r["Job_Title"].ToString(),
                ChapterID = r["ChapterID"].ToString(),
                ShipmentDate = (DateTime)r["ShipmentDate"]
            });
        }
        return Json(list);
    }

    [HttpPost]
    public JsonResult UpdateShipmentDate(int shipmentId, DateTime shipmentDate)
    {
        SqlParameter[] p =
        {
            new("@ShipmentID", shipmentId),
            new("@ShipmentDate", shipmentDate)
        };

        _db.ExecuteProcedure("usp_UpdateShipmentDate", p);
        return Json(new { success = true });
    }
}

PROJECT OVERVIEW (ASSESSMENT 3 – VENDOR SHIPMENT MANAGEMENT)
-----------------------------------------------------------
This project is an ASP.NET MVC application used to manage shipment details
based on vendor and job title selection. The application displays vendors
in a jQuery DataTable, dynamically loads related job titles, and allows
updating shipment dates using stored procedures.

The main focus of this assessment is cascading data loading and inline
update functionality.

TECHNOLOGIES USED
-----------------
- ASP.NET Core MVC for UI and controller logic
- SQL Server (LocalDB) for database
- ADO.NET for database access
- jQuery & jQuery DataTable for dynamic UI
- Bootstrap for layout and styling
- AJAX for client-server communication

NUGET PACKAGES INSTALLED
-----------------------
- Microsoft.Data.SqlClient
  Used to connect the MVC application with SQL Server.

- System.Data.SqlClient
  Used for DataTable, SqlCommand, and SqlParameter support.

DATABASE DESIGN
---------------
The database contains two tables:

1. VendorInfo (Master Table)
   - Stores vendor information.
   - Used to populate the main vendor table in UI.

2. ShipmentInfo (Transaction Table)
   - Stores shipment details such as Job Title, Chapter ID, Shipment Date.
   - Linked to VendorInfo using VendorID.

All database operations are done using stored procedures only.

STORED PROCEDURES USED
---------------------
- usp_GetVendors
  Retrieves all vendors.

- usp_GetJobTitlesByVendor
  Retrieves distinct job titles for the selected vendor.

- usp_GetShipmentDetails
  Retrieves shipment details based on vendor and job title.

- usp_UpdateShipmentDate
  Updates shipment date for a selected shipment.

APPLICATION WORKFLOW
--------------------
1. Page Load
   - The Index page loads.
   - Vendor list is fetched using usp_GetVendors.
   - Vendors are displayed in a jQuery DataTable with paging and sorting.

2. Job Title Loading (Cascading Dropdown)
   - Each vendor row has a Job Title dropdown.
   - When the dropdown is focused, an AJAX call retrieves Job Titles
     related to that vendor using usp_GetJobTitlesByVendor.

3. Shipment Detail Display
   - User selects a Job Title.
   - An AJAX call fetches shipment details using usp_GetShipmentDetails.
   - Shipment details are displayed in a second table.

4. Edit & Update Shipment Date
   - Shipment Date is shown with a calendar (date input).
   - User modifies the date and clicks the Update button.
   - An AJAX call sends the updated date to the server.
   - usp_UpdateShipmentDate updates the database record.

DATA FLOW DIAGRAM
-----------------
UI (Index.cshtml)
  → MVC Controller
    → DBHelper (ADO.NET)
      → Stored Procedure
        → SQL Server

        ↓

IMPORTANT BUSINESS LOGIC
------------------------
- Job Titles are loaded dynamically based on VendorID.
- Shipment details depend on both VendorID and Job Title.
- Shipment Date is editable using a calendar control.
- Paging and sorting are enabled using jQuery DataTable.
- All updates use stored procedures.

SECURITY & BEST PRACTICES
------------------------
- Parameterized SQL prevents SQL injection.
- Database access is centralized using stored procedures.
- Clear separation between UI, controller, and database layers.

SUMMARY
-------
This application demonstrates cascading dropdowns, master-detail data
display, and inline data updates using ASP.NET MVC, jQuery DataTable,
and stored procedures. The design matches real-world enterprise
workflow patterns.
